# Proof Artifacts

## Structure

- `CURRENT_PROOF_SUMMARY.md` — Human-readable current metrics and pass conditions.
- `current/` — Latest regenerated proof bundle.
  - `simworld_summary.json` — Main strict proof summary for current NL/SimWorld proof run.
  - `replay_report.json` — Event-log replay and idempotence report, including scaffold counters.
  - `evidence_integrity_report.json` — SHA-256 evidence-vault integrity report for proof-generated vault entries.
  - `long_horizon_report.json` — Multi-episode deterministic runner report; not proof of long-term learning.
  - `nl_benchmark_report.json` — Natural-language diagnostic scenario report; not proof of broad NL reasoning.
- `history/` — Older proof snapshots for comparison (not current truth).

## Regeneration

```bash
cd global-workspace-runtime-rs
cargo run -p runtime-cli -- proof --strict --long-horizon --nl --out ../artifacts/proof/current
```

## Interpretation

- **current/** files are regenerated by the proof command and should be
  regenerated by CI for live verification.
- **history/** files are preserved for comparison only — they may not
  reflect current runtime behavior.
- The `action_match_rate` in simworld_summary.json is **informational**.
  Proof pass/fail depends on `resource_survival > 0.70`,
  `unsafe_action_count == 0`, `mean_total_score > 0.45`, and
  `evidence_integrity.all_valid == true`.
- Oracle mode is never used — the evaluator selects actions through
  RuntimeLoop, not `expected_action`.
- **Evidence**: Replay counters (evidence_entries: 20) prove event/reducer
  handling for per-cycle EvidenceStored events with real content hashes.
  The evidence integrity report separately validates a small proof-generated
  SHA-256 hash-chain vault. Current proof does not reconstruct a full
  persisted evidence vault from every runtime EvidenceStored event, and
  runtime answers do not yet cite evidence IDs as a full answer-grounding
  system.
