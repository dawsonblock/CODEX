# Proof Artifacts

## Structure

- `CURRENT_PROOF_SUMMARY.md` — Human-readable current metrics and pass conditions.
- `current/` — Latest regenerated proof bundle.
  - `simworld_summary.json` — Scorecard + full trace data.
  - `replay_report.json` — Replay verification with idempotency checks.
- `history/` — Older proof snapshots for comparison (not current truth).

## Regeneration

```bash
cd global-workspace-runtime-rs
cargo run -p runtime-cli -- proof --strict --out ../artifacts/proof/current
```

## Interpretation

- **current/** files are regenerated by the proof command and should be
  regenerated by CI for live verification.
- **history/** files are preserved for comparison only — they may not
  reflect current runtime behavior.
- The `action_match_rate` in simworld_summary.json is **informational**.
  Proof pass/fail depends on `resource_survival > 0.70`,
  `unsafe_action_count == 0`, and `mean_total_score > 0.45`.
- Oracle mode is never used — the evaluator selects actions through
  RuntimeLoop, not `expected_action`.
