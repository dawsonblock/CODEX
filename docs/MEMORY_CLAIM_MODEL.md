# Memory, Claim, and Evidence Model

**Defines the semantic structures for storing, retrieving, and validating claims.**

---

## Memory Record Schema

### **Core Fields**

```rust
pub struct MemoryRecord {
    // Identification
    pub id: String,                          // UUID or hash-based ID
    pub kind: MemoryKind,                    // episodic | semantic | procedural | identity | evidence | contradiction
    
    // Content
    pub text: String,                        // Primary content
    pub entities: Vec<String>,               // Tags: named entities mentioned
    pub tags: Vec<String>,                   // Custom tags for categorization
    
    // Provenance & Confidence
    pub source_ref: String,                  // Where this came from (e.g., "claim_id_X", "external_event_Y")
    pub confidence: f32,                     // 0.0-1.0, credibility estimate
    pub salience: f32,                       // 0.0-1.0, importance/relevance
    pub source_quality: SourceQuality,       // AI-generated | user-provided | external-evidence | verified
    
    // Validity Windows
    pub created_at: SystemTime,              // When record was first stored
    pub updated_at: SystemTime,              // Last modification time
    pub valid_from: Option<SystemTime>,      // When claim became valid (temporal)
    pub valid_to: Option<SystemTime>,        // When claim expires (temporal)
    
    // Status & Lifecycle
    pub status: MemoryStatus,                // active | stale | disputed | superseded | rejected | archived
    
    // Cross-References
    pub evidence_links: Vec<String>,         // Evidence IDs that support this memory
    pub contradiction_links: Vec<String>,    // Memory IDs that contradict this one
    pub superseded_by: Option<String>,       // If superseded, which record replaces it
    
    // Access Tracking
    pub last_retrieved_at: Option<SystemTime>,
    pub retrieval_count: u32,                // How many times retrieved
}

pub enum MemoryKind {
    Episodic,      // Specific events ("I observed X at time T")
    Semantic,      // General knowledge ("fact: X is true")
    Procedural,    // How-to ("procedure: do X then Y")
    Identity,      // Self-model ("I am Y", "my role is Z")
    Evidence,      // Linked evidence ("see evidence ID X")
    Contradiction, // Contradiction tracking ("contradicts memory ID Y")
}

pub enum MemoryStatus {
    Active,        // Current valid memory
    Stale,         // Valid but aged (valid_to passed)
    Disputed,      // Contradicted by another memory
    Superseded,    // Replaced by newer version
    Rejected,      // Marked as false/invalid
    Archived,      // Still in vault but not retrieved by default
}

pub enum SourceQuality {
    AiGenerated,       // Generated by cognitive pipeline (lower priority)
    UserProvided,      // Provided by user (high priority)
    ExternalEvidence,  // From external source (medium priority, may be stale)
    Verified,          // User verified this memory is correct (highest priority)
}
```

## Memory Retrieval Interface

### **Query Contract**

```rust
pub struct MemoryQuery {
    // Filter criteria
    pub text_query: Option<String>,          // Keyword search (default: substring)
    pub entities: Option<Vec<String>>,       // Must mention these entities
    pub kinds: Option<Vec<MemoryKind>>,      // Filter by kind(s)
    pub min_confidence: Option<f32>,         // Minimum confidence threshold
    pub min_salience: Option<f32>,           // Minimum salience threshold
    pub status_filter: Vec<MemoryStatus>,    // Default: [Active] (exclude stale/rejected/disputed)
    
    // Temporal filters
    pub after: Option<SystemTime>,           // Only memories updated after this time
    pub before: Option<SystemTime>,          // Only memories updated before this time
    pub only_valid_at: Option<SystemTime>,   // Only valid at this timestamp
    
    // Options
    pub include_disputed: bool,              // Include disputed claims (default: false)
    pub include_stale: bool,                 // Include stale claims (default: false)
    pub include_superseded: bool,            // Include superseded claims (default: false)
    pub include_rejected: bool,              // Include rejected claims (default: false)
    
    // Limits
    pub limit: Option<usize>,                // Max results to return (default: 10)
    pub order_by: OrderBy,                   // How to rank results
}

pub enum OrderBy {
    RelevanceScore(TextSimilarity),  // Semantic/keyword similarity
    Salience,                         // By salience (importance)
    Confidence,                       // By confidence level
    Recency,                          // By last_updated (newest first)
    RetrievalFrequency,              // By how often retrieved
}

pub struct MemoryHit {
    pub record: MemoryRecord,
    pub score: f32,                  // Relevance/match score (0.0-1.0)
    pub match_type: MatchType,       // How it matched
    pub match_reason: String,        // Why it matched ("keyword: 'weather' found in text")
    pub evidence_refs: Vec<String>,  // Evidence supporting this hit
    pub warning: Option<String>,     // "stale", "disputed", "weak_source", etc.
}

pub enum MatchType {
    ExactKeyword,      // Query term found in text
    PartialKeyword,    // Query term partially found
    EntityMatch,       // Entity mentioned in memory
    KindMatch,         // Kind filter matched
    TemporalMatch,     // Within temporal window
}
```

### **MemoryProvider Trait**

```rust
#[async_trait]
pub trait MemoryProvider: Send + Sync {
    async fn retrieve(&self, query: &MemoryQuery) 
        -> Result<Vec<MemoryHit>, MemoryError>;
    
    async fn store(&mut self, record: MemoryRecord) 
        -> Result<String, MemoryError>;  // Returns record ID
    
    async fn update(&mut self, id: &str, updates: MemoryUpdateRequest) 
        -> Result<(), MemoryError>;
    
    async fn mark_stale(&mut self, id: &str) 
        -> Result<(), MemoryError>;
    
    async fn mark_disputed(&mut self, id: &str, contradicts: &str) 
        -> Result<(), MemoryError>;
    
    async fn mark_rejected(&mut self, id: &str, reason: &str) 
        -> Result<(), MemoryError>;
    
    async fn get_by_id(&self, id: &str) 
        -> Result<Option<MemoryRecord>, MemoryError>;
    
    async fn list_all(&self) 
        -> Result<Vec<MemoryRecord>, MemoryError>;

    async fn purge_timestamp(&mut self, before: SystemTime, keep_archived: bool)
        -> Result<u32, MemoryError>;  // Returns count purged
}
```

## Claim Lifecycle Model

### **Claim Record Schema**

```rust
pub struct Claim {
    // Identification
    pub id: String,                          // UUID
    pub subject: String,                     // "X"
    pub predicate: String,                   // "is"
    pub object: String,                      // "Y"
    // Together: "X is Y"
    
    // Confidence & Source
    pub confidence: f32,                     // 0.0-1.0, belief strength
    pub status: ClaimStatus,                 // active | disputed | superseded | rejected | archived
    pub source_quality: SourceQuality,       // Where it came from
    pub extraction_confidence: f32,          // Confidence in extraction (vs. confidence in claim)
    
    // Corroboration & Contradiction
    pub corroboration_count: u32,            // How many times verified/supported
    pub contradiction_count: u32,            // How many contradictions
    pub contradicted_by: Vec<String>,        // Contradiction record IDs
    pub superseded_by: Option<String>,       // If replaced, which claim replaces it
    
    // Validity & Lifecycle
    pub valid_from: SystemTime,
    pub valid_to: Option<SystemTime>,
    pub created_at: SystemTime,
    pub updated_at: SystemTime,
    
    // Cross-References
    pub evidence_links: Vec<String>,         // Evidence supporting this claim
    pub memory_links: Vec<String>,           // Memory records supporting this claim
    pub audit_trail: Vec<ClaimAuditEvent>,   // History of transformations
}

pub enum ClaimStatus {
    Active,       // Currently believed claim
    Disputed,     // Has contradicting claims (both stored, decision deferred)
    Superseded,   // Replaced by newer claim (still stored for history)
    Rejected,     // Marked as false (not used in answers unless context requires)
    Archived,     // Old claim kept for history but not retrieved
}

pub struct ClaimAuditEvent {
    pub timestamp: SystemTime,
    pub event_type: ClaimEventType,
    pub reason: String,
    pub actor: String,  // "user", "system", "cognitive_pipeline", etc.
}

pub enum ClaimEventType {
    Created,
    CorroboratedBy { other_claim_id: String },
    ContradictedBy { contradiction_claim_id: String },
    MarkedDisputed,
    MarkedRejected,
    ArchivedDueToAge,
    MarkedSupersedBy { new_claim_id: String },
}
```

### **Claim Lifecycle Rules**

#### **Rule 1: Corroboration Increases Confidence**

```rust
if claim_matches_pattern(&new_evidence, &existing_claim) {
    existing_claim.corroboration_count += 1;
    
    // Boost confidence (but never past 1.0)
    let boost = 0.05 * (new_evidence.confidence / 10.0);
    existing_claim.confidence = (existing_claim.confidence + boost).min(1.0);
    
    // Boost salience (it matters more when verified)
    log_audit_event(claim_id, ClaimEventType::CorroboratedBy { ... });
    memory.store(existing_claim);
}
```

#### **Rule 2: Contradiction Creates Dispute, Not Silent Overwrite**

```rust
if contradiction_detected(&claim_A, &claim_B) {
    // Mark both as disputed (don't delete A)
    claim_A.status = ClaimStatus::Disputed;
    claim_B.status = ClaimStatus::Disputed;
    claim_A.contradicted_by.push(claim_B.id.clone());
    claim_B.contradicted_by.push(claim_A.id.clone());
    
    // Store both, emit audit events
    log_audit_event(claim_A.id, ClaimEventType::ContradictedBy { claim_B.id });
    log_audit_event(claim_B.id, ClaimEventType::ContradictedBy { claim_A.id });
    
    memory.store(claim_A);
    memory.store(claim_B);
}
```

#### **Rule 3: Supersession Preserves Old Claims**

```rust
if claim_B_replaces_claim_A(&claim_B, &claim_A) {
    // Mark A as superseded, don't delete it
    claim_A.status = ClaimStatus::Superseded;
    claim_A.superseded_by = Some(claim_B.id.clone());
    
    // Mark B as active
    claim_B.status = ClaimStatus::Active;
    
    // Store both
    log_audit_event(claim_A.id, ClaimEventType::MarkedSupersedBy { claim_B.id });
    memory.store(claim_A);
    memory.store(claim_B);
}
```

#### **Rule 4: Confidence Does NOT Decay Over Time**

```rust
// WRONG: claim.confidence -= 0.01 every day (no, confidence doesn't decay)
// RIGHT: Only salience/recency decay (memory becomes less salient)

if claim.updated_at.elapsed() > Duration::days(30) {
    // Claim becomes stale (optional: mark status::Stale)
    // But confidence is unchanged (it was what we believed)
    log_audit_event(claim.id, "marked stale after 30 days");
}
```

#### **Rule 5: Rejected Claims Excluded from Context**

```rust
let query = MemoryQuery { 
    status_filter: vec![Active, Disputed],  // Does NOT include Rejected
    include_rejected: false,                 // Explicit: don't retrieve rejected
    ..Default::default()
};

let hits = memory.retrieve(&query).await?;
// Rejected claims never appear in hits (unless explicitly requested)

// When building answer, filter out rejected and disputed (unless needed for reasoning)
```

#### **Rule 6: AI-Generated Claims Have Lower Source Quality**

```rust
let claim = Claim {
    source_quality: SourceQuality::AiGenerated,  // Lower priority
    confidence: score,  // Use cognitive score
    extraction_confidence: certainty,  // Separate from belief confidence
    
    // Corroboration with user input can promote it
    // Still marked as AI-generated for provenance tracing
    ..Default::default()
};

memory.store(claim)?;
```

#### **Rule 7: Weak-Evidence Claims Locked Below Threshold**

```rust
if claim.confidence < 0.5 {
    // Cannot promote above 0.5 without external evidence
    claim.status = ClaimStatus::Active;  // Still active (believe it tentatively)
    claim.confidence = claim.confidence.min(0.5);  // Capped at 0.5
    
    log_audit_event(claim.id, "confidence capped at 0.5 due to weak evidence");
    memory.store(claim)?;
}
```

## Contradiction Engine Integration

### **Contradiction Detection**

```rust
pub struct ContradictionDetector;

impl ContradictionDetector {
    pub fn detect(claim_a: &Claim, claim_b: &Claim) -> Option<Contradiction> {
        // Check for structural conflict
        if same_subject_predicate(&claim_a, &claim_b) && 
           conflicting_objects(&claim_a.object, &claim_b.object) {
            return Some(Contradiction {
                claim_a_id: claim_a.id.clone(),
                claim_b_id: claim_b.id.clone(),
                conflict_type: ConflictType::ObjectConflict,
                severity: calculate_severity(claim_a, claim_b),
            });
        }
        None
    }
}

pub struct Contradiction {
    pub claim_a_id: String,
    pub claim_b_id: String,
    pub conflict_type: ConflictType,
    pub severity: f32,  // 0.0-1.0, higher = more serious
}

pub enum ConflictType {
    ObjectConflict,        // "X is A" vs. "X is B" where A != B
    TemporalConflict,      // Claims about different times
    SourceConflict,        // Different sources say different things
    LogicalConflict,       // "A and not A"
}
```

### **Contradiction Handling in Answer Builder**

```rust
fn build_answer(&self, query: &str, claims: Vec<Claim>) -> AnswerEnvelope {
    let disputed_claims = claims.iter()
        .filter(|c| c.status == ClaimStatus::Disputed)
        .collect::<Vec<_>>();
    
    if !disputed_claims.is_empty() {
        warnings.push(format!(
            "Answer uses disputed claims (contradictions found): {:?}",
            disputed_claims.iter().map(|c| c.id.clone()).collect::<Vec<_>>()
        ));
        
        // Still use the claim (don't drop it), but warn
        // Let human decide how to weight it
    }
    
    let rejected_claims = claims.iter()
        .filter(|c| c.status == ClaimStatus::Rejected)
        .collect::<Vec<_>>();
    
    // Rejected claims: try not to use them (except in "why I rejected this" context)
    
    AnswerEnvelope {
        text: build_text(claims),
        confidence: calculate_confidence(claims),
        warnings,
        cited_claim_ids: claims.iter().map(|c| c.id.clone()).collect(),
        ..Default::default()
    }
}
```

## Evidence-Memory-Claim Flow

### **Typical Flow**

1. **Evidence Collected** — Observation or reasoning produces evidence
   - Evidence stored in evidence vault (SHA-256 hash chain)
   - Evidence assigned ID, tagged with source

2. **Claim Generated** — Evidence triggers claim generation
   ```rust
   let claim = Claim {
       subject: extract_subject(&evidence),
       predicate: "is",
       object: extract_object(&evidence),
       evidence_links: vec![evidence_id],
       source_quality: SourceQuality::AiGenerated,
       confidence: evidence.quality * reasoning_certainty,
       ..Default::default()
   };
   ```

3. **Claim Stored** — Claim added to memory
   ```rust
   memory.store(claim)?;
   ```

4. **Retrieval Query** — User query triggers memory retrieval
   ```rust
   let query = MemoryQuery {
       text_query: Some(user_query),
       kinds: Some(vec![Semantic]),
       status_filter: vec![Active],
       include_disputed: false,
       include_rejected: false,
       ..Default::default()
   };
   let hits = memory.retrieve(&query).await?;
   ```

5. **Contradiction Check** — New query results checked against stored claims
   ```rust
   for hit in &hits {
       for stored_claim in &stored_claims {
           if let Some(contradiction) = detector.detect(&hit.record, stored_claim) {
               // Mark both as disputed
           }
       }
   }
   ```

6. **Answer Building** — Hits + claims used to build response
   ```rust
   let answer = answer_builder.build(
       &user_query,
       hits.iter().map(|h| h.record.clone()).collect(),
       &contradiction_engine,
   );
   ```

## Default Retrieval Behavior

### **Safe Defaults Based on Status**

```rust
impl MemoryQuery {
    pub fn for_answer_building(text: &str) -> Self {
        MemoryQuery {
            text_query: Some(text.to_string()),
            status_filter: vec![
                MemoryStatus::Active,           // ✅ Yes
                MemoryStatus::Disputed,         // ⚠️ Yes (warn user)
            ],
            include_disputed: false,             // But filter by status above
            include_stale: false,                // ❌ No (marked stale, excluded)
            include_superseded: false,           // ❌ No (replaced by newer)
            include_rejected: false,             // ❌ No (user rejected)
            kind_filter: None,                   // All kinds (episodic, semantic, etc.)
            min_confidence: 0.3,                 // Include tentative beliefs
            order_by: OrderBy::Salience,         // Sorted by importance
            limit: Some(10),
            ..Default::default()
        }
    }
    
    pub fn for_full_audit() -> Self {
        MemoryQuery {
            // Include everything, for debugging/audit purposes
            include_disputed: true,
            include_stale: true,
            include_superseded: true,
            include_rejected: true,
            status_filter: vec![
                Active, Disputed, Stale, Superseded, Rejected, Archived
            ],
            ..Default::default()
        }
    }
}
```

## Testing Requirements

### **Memory Backend Tests**

- ✅ Can store and retrieve records
- ✅ Keyword matching works
- ✅ Entity filtering works
- ✅ Status filtering excludes rejected by default
- ✅ Status filtering excludes stale by default (unless requested)
- ✅ Status filtering includes disputed by default (but marked with warning)
- ✅ Temporal filtering works (valid_from, valid_to, after, before)
- ✅ Evidence links preserved
- ✅ Contradiction links tracked
- ✅ Deterministic ordering (no randomness)
- ✅ No duplicates returned

### **Claim Lifecycle Tests**

- ✅ Corroboration updates confidence and salience
- ✅ Contradiction marks both claims as disputed (doesn't delete)
- ✅ Supersession preserves old claim, creates new active claim
- ✅ Rejection excludes claim from answer context
- ✅ Stale claims not retrieved unless requested
- ✅ Confidence does not decay
- ✅ Audit trail recorded for all transitions
- ✅ Weak evidence claims capped at confidence 0.5

### **Answer Builder Tests**

- ✅ Answer includes cited_claim_ids
- ✅ Disputed claim triggers warning
- ✅ Rejected claim excluded (unless debug context)
- ✅ Weak-evidence claim deferred or lowered confidence
- ✅ No raw internal state leaks to user answer
